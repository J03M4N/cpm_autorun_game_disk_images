Terminal: dumb terminal or VT52. (Steven Flintham's 'STEM' emulates VT52. Search web for 'VT100/ANSI terminal emulation via OSWRCH'.)

CPMtools diskdef:

diskdef acorn3a ssd
seclen 512
tracks 160
sectrk 5
blocksize 2048
maxdir 128
skewtab 0,2,4,1,3
boottrk 3
os 2.2
end

Source boot disk from MAME software list.

Workflow: Source .SSD image can be loaded and edited with CPMtoolsGUI, No conversion required. 

Autostart: Support for 'SUBMIT.COM' with 'BOOT.SUB' is built-in. Hex edit disk image, search for current autorun phrase, make sure to terminate with 0D 0A.

Emulation: .SSD can only be halfway full, not sure why. 

MAME276 command line:

mame bbcb -flop1 "[path to CP/M bootdisk image]" -tube z80 -fdc acorn1770 -skip_gameinfo -window -nomaximize 

MAME276 command line for STEM (VT52 emulation):

mame bbcb -flop1 "[path to 'stem.ssd' image]" -ui_active -skip_gameinfo -window -tube z80 -fdc acorn1770 -internal swr128 -autoboot_delay 4 -autoboot_command "*srload :0.$.stem 8000 z q\n"

After STEM finishes loading, use MAME UI 'File Manager' to replace 'stem.ssd' in floppydisk1 to whichever CPM autorun game image you wish to play, select 'Read Only'. Exit UI.

Beebem422:

"File Menu','Load State', use 'CPM.uefstate' from 'bbcCPMgames' folder, disable 'Write Protect' in 'Disk Options', 'Load Disk 0' with CPM autorun game image, press return (or 'Reset' in 'File Menu').

Beebem422 for STEM (vt52 emulation):

"File Menu','Load State', use 'STEMvt52.uefstate' from 'bbcCPMgames/STEM' folder, disable 'Write Protect' in 'Disk Options', 'Load Disk 0' with CPM autorun game image, press return.
