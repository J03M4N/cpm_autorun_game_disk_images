Source boot disk image from MAME software list.

Workflow: load .td0 (or.IMD) in HxC, export raw .IMG, edit with CPMtoolsGUI, load RAW image in HxC, export to .IMD. 

Autostart: This system runs autostart at warmboot, so it uses a clever scheme for autostart. To implement this from the bootdisk, you must remount disk through UI, choose 'write to separate disk' and choose '.MFM', then use HxC to load .MFM, and export raw '.IMG', then use standard workflow, or you can hexedit at location 0x3360.

Character set is messed up in MAME emulation.

*MAME276 command line:

MAME alphatro -flop1 "[path to cpm bootdisk]" -window -nomaximize -skip_gameinfo -autoboot_delay 5 -autoboot_command 1

*This system defaults to no FDD installed, so the first time you run the system, invoke user interface, select 'Machine Configuration', change 'FDD unit installed' to "YES" and select 'Reset System', ESC to exit. This will create a '.cfg' file in MAME. This only needs to be once. CPM will now boot.
